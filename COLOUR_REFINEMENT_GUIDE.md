# Colour Refinement Guide for The Season Edit

## Purpose
This document provides the methodology and logic used to refine seasonal colour palettes from 30+ candidate colours down to the final 36-colour extended palette (6×6 grid).

---

## Methodology

### 1. Warm vs Cool Analysis
**Use RGB hex values to determine undertones:**
- **Cool = More Blue than Red** (R < B in hex)
  - Example: #96D8C8 → R=150, B=200 → Cool (blue-dominant)
- **Warm = More Red than Blue** (R > B in hex)
  - Example: #A8D96B → R=168, B=107 → Warm (red-dominant, yellow-green)

**Key principle:** Always analyze hex values objectively, don't rely on colour names alone (e.g., "Warm Periwinkle" was actually cool when analyzed).

### 2. Season Characteristics

#### Light Spring
- **Primary:** Light (delicate, not heavy)
- **Secondary:** Warm (peachy, golden undertones)
- **Tertiary:** Clear (not muted, but softer than Bright Spring)
- **Avoid:** Deep/dark colours, cool blues, muddy/muted tones

**Ideal hex characteristics:**
- High overall brightness (R+G+B values)
- Warm undertones (R ≥ B in most colours)
- Clear saturation (not greyed out)

### 3. Redundancy Rules

**Two colours are redundant if:**
1. **Visually indistinguishable** in a grid (too similar in value/saturation)
2. **Hex values differ by < 20 points** across R, G, B
3. **Same colour family + same tone** (e.g., two pale peachy-beiges)

**Keep variety by ensuring:**
- Value range (light → medium tones for Light Spring)
- Hue diversity within colour families
- Practical fashion colour names

### 4. Colour Family Balance

**Target distribution for 36 colours:**
- Pinks/Corals/Reds: 8-10 (Light Spring signature)
- Oranges: 3-4
- Yellows: 4-5
- Greens: 5-6
- Blues/Aquas: 4-5
- Purples/Mauves: 3-4
- Neutrals: 6-8 (essential for wardrobe building)

## Key Principles for Other Seasons

### When refining other seasons:
1. **Always analyze hex values** for warm/cool determination (R vs B)
2. **Compare candidate colours side-by-side** visually in Canva/design tool
3. **Check for redundancy** - if two colours look too similar in a grid, pick one
4. **Prioritize common fashion names** (cream > oyster, camel > mushroom)
5. **Maintain colour family balance** - don't overload one family
6. **Include 6-8 neutrals** - they're essential for practical wardrobes
7. **Respect season characteristics:**
   - **Light seasons:** Light-to-medium values only
   - **Deep seasons:** Medium-to-dark values
   - **Warm seasons:** R > B in most colours
   - **Cool seasons:** B > R in most colours
   - **Soft seasons:** Muted, greyed-out colours
   - **Bright/Clear seasons:** High saturation, no muddiness

### Where rejected colours go:
- **Too pale for Light Spring** → Light Summer
- **Too cool for Light Spring** → Light Summer, Cool Summer
- **Too saturated/deep for Light Spring** → Warm Spring, Bright Spring
- **Too muted for Light Spring** → Soft Autumn, Soft Summer
- **Warm but deeper than Light Spring** → Warm Spring, Warm Autumn

## Output Format for Colour Analysis

When analyzing colours, use this format:

```
✅ **Colour Name #HEX** - description (light/dark, warm/cool, clear/muted) - KEEP - reason
❌ **Colour Name #HEX** - description - Skip - reason (→ better season)
⚠️ **Colour Name #HEX** - description - Maybe - caveat
```

**Example:**
```
✅ **Warm Turquoise #76D7C4** - warmest option, green-dominant, perfect for Light Spring - KEEP
❌ **Light Turquoise #77D9D6** - cool, blue-green, too cool for Light Spring - Skip (→ Cool Summer)
```

## Grid Layout
- **6 columns × 6 rows = 36 colours**
- Organize by colour family in rainbow order: pinks → oranges → yellows → greens → blues → purples → neutrals
- Keep related colours together for visual flow
